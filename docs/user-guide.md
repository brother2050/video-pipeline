# VideoPipeline 使用指南

## 1. 系统要求

- Python 3.11+
- Node.js 18+
- FFmpeg
- 磁盘空间 >= 10GB
- 可选：NVIDIA GPU + CUDA（用于本地 AI 模型）

## 2. 安装步骤

```bash
# 检查环境
python scripts/check_env.py

# 一键安装
python scripts/install.py

# 启动
python scripts/start.py
```

启动后访问 http://localhost:5173

## 3. 快速开始

### 3.1 创建项目
1. 点击"新建项目"
2. 填写名称、描述、题材类型
3. 设置目标集数和每集时长
4. 点击创建

### 3.2 配置供应商
1. 进入"供应商"页面
2. 为每种能力配置至少一个供应商
3. 填写 API Key（如果需要）
4. 点击"测试连接"验证
5. 保存配置

### 3.3 逐阶段创作
1. 进入项目，看到 9 阶段时间线
2. 点击当前阶段进入审核页
3. 编辑提示词（可选）
4. 点击"生成"，等待 AI 生成候选
5. 从候选中选择最好的
6. 点击"通过并锁定"
7. 自动进入下一阶段

## 4. 功能详解

### 4.1 审核流程
- 每个阶段生成多个候选
- 选择最佳候选后点击"通过"
- 不满意可以"拒绝"或"重新生成"
- 任何时候可以"回退"到前面的阶段

### 4.2 节点管理
- 添加本地或远程节点
- 设置节点能力和模型
- 实时查看节点状态和心跳

### 4.3 ComfyUI 集成
1. 在供应商配置中选择"图像"或"视频" Tab
2. 添加 ComfyUI 供应商，填写地址（如 http://localhost:8188）
3. 点击"导入 ComfyUI 工作流 JSON"上传工作流文件
4. 系统自动分析工作流结构，显示可覆盖的参数
5. 确认后保存

### 4.4 SD WebUI 集成
1. 启动 SD WebUI（带 --api 参数）
2. 在供应商配置中添加 SD WebUI 供应商
3. 填写地址（默认 http://localhost:7860）
4. 测试连接

## 5. 常见问题

**Q: 生成很慢怎么办？**
A: 可以配置多个供应商，远程失败自动降级到本地。本地 GPU 越强，生成越快。

**Q: 回退会丢失数据吗？**
A: 不会。所有版本历史保留，回退只是重置阶段状态。

**Q: 可以同时运行多个项目吗？**
A: 可以。每个项目独立的阶段状态。

## 6. 配置参考

配置文件位于 config/ 目录：
- default.yaml: 服务端口、数据库、心跳
- nodes.yaml: 节点列表
- suppliers.yaml: 供应商配置
